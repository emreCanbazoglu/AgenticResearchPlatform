# Task 01 — Fetch Real Binance OHLCV Data

## Status
Pending

## Owner
Codex agent

## Context

`data/trading/btc_usdt_1d.csv` and `data/trading/eth_usdt_1d.csv` currently contain 1000 rows of seeded random-walk data generated by `data/trading/generate_crypto_data.py`. These are meaningless for research — strategy scores cannot be interpreted.

The Binance public REST API returns real historical OHLCV data without authentication:
```
GET https://api.binance.com/api/v3/klines?symbol=BTCUSDT&interval=1d&limit=1000
```

Response format: array of arrays — each element is:
```
[open_time, open, high, low, close, volume, close_time, quote_volume, trade_count, ...]
```

## Task

### 1. Create `scripts/fetch_crypto_data.py`

Write a standalone script that:
1. Fetches 1000 daily candles for `BTCUSDT` and `ETHUSDT` from the Binance klines endpoint
2. Saves them as `data/trading/btc_usdt_1d.csv` and `data/trading/eth_usdt_1d.csv`
3. Overwrites existing files

**CSV format** (must match what `TradingAdapter._load_close_prices()` expects — it reads the `close` column):
```
open_time,open,high,low,close,volume
1609459200000,29000.0,29300.0,28800.0,29100.0,5000.0
...
```

**Implementation requirements:**
- Standard library only: `urllib.request`, `json`, `csv`, `pathlib`
- No `requests`, `pandas`, `ccxt`, or other third-party libraries
- Print progress: `"Fetched 1000 rows for BTCUSDT"` etc.
- Exit with a non-zero status code and clear error message if the API call fails (network error, HTTP error, or malformed response)
- Handle the case where `data/trading/` directory does not exist (create it)

**Script interface:**
```bash
uv run python scripts/fetch_crypto_data.py
# Fetched 1000 rows for BTCUSDT → data/trading/btc_usdt_1d.csv
# Fetched 1000 rows for ETHUSDT → data/trading/eth_usdt_1d.csv
```

**Fields to include in CSV (in this order):**
- `open_time` — millisecond Unix timestamp (index 0 of klines response)
- `open` — index 1
- `high` — index 2
- `low` — index 3
- `close` — index 4
- `volume` — index 5
